# Toggle — Code

The `<Toggle>` chassis at `src/lib/components/chassis/forms/Toggle.svelte` is a thin wrapper around Carbon Design System's Toggle primitive. It bridges Carbon's `on:toggle` `CustomEvent<{toggled: boolean}>` to a Svelte 5 `onToggle(toggled: boolean)` callback (matching the existing chassis pattern in `forms/Tabs.svelte`).

## Public API — `<Toggle>` component

```typescript
interface Props {
  toggled?: boolean;        // bindable
  labelText: string;        // REQUIRED — Carbon a11y discipline
  labelA?: string;          // off-state inline label (default: 'Off')
  labelB?: string;          // on-state inline label (default: 'On')
  hideLabel?: boolean;      // visually hide the heading label (still in the a11y tree)
  disabled?: boolean;
  size?: 'default' | 'sm';
  name?: string;
  id?: string;
  class?: string;
  onToggle?: (toggled: boolean) => void;
}
```

| Prop | Type | Default | Description |
|---|---|---|---|
| `toggled` | `boolean` | `false` | Bindable. Drives `aria-checked` and the visual track state |
| `labelText` | `string` | **REQUIRED** | Heading label — accessible name for the toggle |
| `labelA` | `string` | `'Off'` | Inline state label rendered when `toggled === false` |
| `labelB` | `string` | `'On'` | Inline state label rendered when `toggled === true` |
| `hideLabel` | `boolean` | `false` | Visually hide the heading; remains in the a11y tree |
| `disabled` | `boolean` | `false` | Disables click + keyboard activation |
| `size` | `'default' \| 'sm'` | `'default'` | Carbon size variant |
| `name` | `string` | `undefined` | Form field name (for `<form>` submission) |
| `id` | `string` | autogenerated | DOM `id` (Carbon defaults to a stable hash) |
| `class` | `string` | `undefined` | Extra class on the outer Carbon wrapper |
| `onToggle` | `(toggled: boolean) => void` | `undefined` | Callback fired on user toggle. Receives the new value |

## Internal wiring

```svelte
<script lang="ts">
  import { Toggle as CarbonToggle } from 'carbon-components-svelte';

  interface Props { /* ...as above... */ }
  let {
    toggled = $bindable(false),
    labelText,
    labelA = 'Off',
    labelB = 'On',
    hideLabel = false,
    disabled = false,
    size = 'default',
    name,
    id,
    class: extraClass = '',
    onToggle
  }: Props = $props();

  function handle(e: CustomEvent<{ toggled: boolean }>): void {
    toggled = e.detail.toggled;
    onToggle?.(e.detail.toggled);
  }
</script>

<CarbonToggle
  bind:toggled
  {labelText}
  {labelA}
  {labelB}
  {hideLabel}
  {disabled}
  {size}
  {name}
  {id}
  class={extraClass}
  on:toggle={handle}
/>
```

## Carbon → chassis API mapping

| Carbon prop | Chassis prop | Notes |
|---|---|---|
| `toggled` | `toggled` | Identical, both bindable |
| `labelText` | `labelText` | Argos requires (Carbon optional but a11y-essential) |
| `labelA` / `labelB` | `labelA` / `labelB` | Identical |
| `hideLabel` | `hideLabel` | Identical |
| `disabled` | `disabled` | Identical |
| `size` | `size` | Identical |
| `name` / `id` / `class` | `name` / `id` / `class` | Identical |
| `on:toggle` `CustomEvent` | `onToggle(toggled: boolean)` | Bridge — Argos chassis convention |

## Paste-ready snippets

### Minimal (heading-only label)

```svelte
<Toggle bind:toggled={enabled} labelText="Enable feature X" />
```

### With explicit on / off names

```svelte
<Toggle
  bind:toggled={connected}
  labelText="Server connection"
  labelA="Disconnected"
  labelB="Connected"
/>
```

### Hidden heading (icon-style toggle)

```svelte
<Toggle
  bind:toggled={advanced}
  labelText="Show advanced controls"
  hideLabel
/>
```

### With change callback

```svelte
<Toggle
  bind:toggled={config.shouldConnectOnStartup}
  labelText="Connect on startup"
  onToggle={(t) => persistSetting('connectOnStartup', t)}
/>
```

## What the chassis does NOT expose

- `ref` / `refLabel` — bind to the Carbon component directly if needed (advanced).
- `on:click` / `on:focus` / `on:blur` / `on:keyup` — pass via `class` + native DOM listeners on the parent wrapper. Most consumers don't need them.

## File budget

The chassis is ~70 LOC including imports + JSDoc. Architecture rule: ≤ 300 LOC/file, ≤ 50 LOC/function. `handle()` is 4 LOC.

## Tests
